XSS is a web vulnerability where an attacker injects malicious JavaScript into a website, and that script executes in other users' browsers.

# Why is XSS dangerous?

- Steal cookies
- Hijack sessions
- Show fake login forms
- Take over accounts
- Modify website content
- Execute arbitrary JavaScript in victim’s browser

# Basic Example

Suppose a website has a comment box.

An attacker submits:

<script>alert(1)</script>

If the website displays it without sanitization, the browser executes it:

Result:

alert(1)

# How XSS Works

1. Website accepts user input
2. Input is not properly sanitized or escaped
3. Input is reflected or stored in page
4. Browser interprets it as HTML/JavaScript
5. Script executes

# How to Detect XSS

Start by testing special characters:

< > " ' /

Then try simple payloads:

<script>alert(1)</script>
or
<img src=x onerror=alert(1)>

If JavaScript executes, the page is vulnerable.

# Contexts of XSS

HTML Context

<h1>USERINPUT</h1>

Payload:

<script>alert(1)</script>

Attribute Context

<input value="USERINPUT">

Payload:

"><script>alert(1)</script>

JavaScript Context

var x = "USERINPUT";

Payload:

";alert(1);//

# Types of XSS

1. Reflected XSS

The payload is reflected immediately in the server response.

Example URL:

https://site.com/search?q=test

Attacker injects:

<script>alert(1)</script>

Final URL:

https://site.com/search?q=<script>alert(1)</script>

If page outputs:

You searched for <script>alert(1)</script>

Common places:

- Search boxes
- Error messages
- URL parameters
- Login pages
- Query strings

2. Stored XSS

The payload is stored permanently in database and executes for every user.

Common targets:

- Comment sections
- Forums
- User profiles
- Chat systems

Attacker stores:

<script>alert(1)</script>

Whenever another user visits the page, the script executes automatically.

3. DOM-Based XSS

The vulnerability exists in frontend JavaScript instead of the server.

Example:

document.getElementById("demo").innerHTML = location.hash;

URL:

site.com/#<img src=x onerror=alert(1)>

The browser inserts the payload into DOM and executes it.

Example Visualization

Reflected:
User → Payload → Server → Response → Browser executes

Stored:
Attacker → Database → Victim opens page → Executes

DOM:
URL/Input → Frontend JS → DOM manipulation → Execution

# Prevention

- Output encoding
- HTML escaping
- CSP (Content Security Policy)
- Input sanitization
- HttpOnly cookies
